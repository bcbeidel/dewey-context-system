# UX Research Methods Catalog

**Purpose**: Comprehensive reference of 20 UX research methods with when to use, how to execute, and sample sizes.

**Version**: 1.0
**Created**: 2026-02-08
**Source**: Nielsen Norman Group + industry best practices

---

## How to Use This Catalog

**By research goal**:
- **Discover user needs** → User Interviews, Contextual Inquiry, Diary Studies
- **Evaluate design** → Usability Testing, Heuristic Evaluation, Cognitive Walkthrough
- **Measure** → Surveys, Analytics Analysis, A/B Testing
- **Understand mental models** → Card Sorting, Tree Testing

**By timeline**:
- **Days** → Heuristic Evaluation, Expert Review, Analytics Analysis
- **1 week** → Usability Testing (5 users), Quick surveys
- **2-4 weeks** → User Interviews, Contextual Inquiry, Comprehensive surveys
- **4+ weeks** → Diary Studies, Field Studies, Longitudinal studies

## Discovery & Exploration Methods

### 1. User Interviews

**What**: One-on-one conversations to understand needs, behaviors, attitudes, pain points.

**When to use**:
- Exploratory research (don't know much yet)
- Understanding motivations and context
- Generating hypotheses for testing

**Duration**: 45-60 minutes per interview

**Sample size**: 5-8 users per user segment

**Process**:
1. Create discussion guide (open-ended questions)
2. Recruit representative users
3. Conduct interviews (record with consent)
4. Analyze for themes and patterns

**Best for**: Attitudinal, qualitative, scripted use

**Deliverable**: Themes, insights, user quotes, opportunity areas

**Pro tip**: Start broad, narrow down based on interesting threads. Use "Tell me about..." not "Do you like..."

---

### 2. Contextual Inquiry

**What**: Observe and interview users in their natural environment while they work.

**When to use**:
- Understand workflows and context
- Discover workarounds and pain points
- See actual behavior (not stated behavior)

**Duration**: 2-3 hours per session

**Sample size**: 6-12 users

**Process**:
1. Arrange to observe in user's environment
2. Use master-apprentice model (they teach you)
3. Observe, then ask clarifying questions
4. Note tools, workarounds, interruptions
5. Take photos/videos (with permission)

**Best for**: Behavioral, qualitative, natural use

**Deliverable**: Workflow maps, pain points, contextual insights, photos

**Pro tip**: Minimize interruptions while observing. Ask questions during natural breaks.

---

### 3. Diary Studies

**What**: Users self-report activities, thoughts, and experiences over days or weeks.

**When to use**:
- Longitudinal behaviors (over time)
- Understanding usage patterns in natural context
- Capturing moments as they happen

**Duration**: 1-4 weeks (user participation)

**Sample size**: 10-20 users

**Process**:
1. Define what to track (activities, pain points, usage)
2. Create diary format (app, form, photo uploads)
3. Recruit and train participants
4. Set reminders for entries
5. Analyze entries for patterns

**Best for**: Behavioral, qualitative/quantitative, natural use

**Deliverable**: Usage patterns, temporal insights, context-specific pain points

**Pro tip**: Make it easy - use mobile apps. Remind participants daily. Incentivize completion.

---

### 4. Field Studies

**What**: Extended observation of users in their natural environment.

**When to use**:
- Deep understanding of work practices
- Cultural and social factors
- Complex workflows and interactions

**Duration**: Days to weeks in field

**Sample size**: 3-10 sites/users

**Process**:
1. Gain access to field sites
2. Observe users over extended period
3. Interview informally during observations
4. Document workflows, culture, practices
5. Synthesize ethnographic insights

**Best for**: Behavioral, qualitative, natural use

**Deliverable**: Ethnographic insights, cultural patterns, workflow documentation

**Pro tip**: Build rapport first. Blend in. Note not just what people do, but why and how context influences behavior.

## Evaluation & Validation Methods

### 5. Usability Testing

**What**: Observe users attempting realistic tasks to identify usability problems.

**When to use**:
- Evaluate specific design or prototype
- Identify usability issues
- Validate design decisions

**Duration**: 45-60 minutes per session

**Sample size**: 5 users (Nielsen research: finds 85% of issues)

**Process**:
1. Define realistic tasks (not instructions)
2. Prepare prototype or product
3. Ask users to complete tasks while thinking aloud
4. Observe and note: success, time, errors, verbalizations
5. Don't help - observe where they struggle
6. Probe after each task

**Best for**: Behavioral, qualitative, scripted use

**Deliverable**: Usability issues, task success rates, severity ratings, recommendations

**Pro tip**: "Try to [goal]" not "Click the button, then...". Silence is powerful - let users struggle.

---

### 6. Heuristic Evaluation

**What**: Expert review of interface against usability principles (heuristics).

**When to use**:
- Quick evaluation without users
- Early design stages
- Complement to usability testing

**Duration**: 2-4 hours per evaluator

**Sample size**: 3-5 expert evaluators

**Process**:
1. Select heuristics (Nielsen's 10, or custom)
2. Have experts independently evaluate interface
3. Each evaluator identifies violations
4. Aggregate and prioritize findings
5. Report issues with severity ratings

**Best for**: Attitudinal (expert), qualitative, scripted use

**Deliverable**: List of usability violations, severity ratings, recommendations

**Pro tip**: Use multiple evaluators. Each finds different issues. Average issues found per evaluator: 35%.

**Nielsen's 10 Heuristics**:
1. Visibility of system status
2. Match system and real world
3. User control and freedom
4. Consistency and standards
5. Error prevention
6. Recognition rather than recall
7. Flexibility and efficiency
8. Aesthetic and minimalist design
9. Help users recognize, diagnose, recover from errors
10. Help and documentation

---

### 7. Cognitive Walkthrough

**What**: Step-through evaluation asking if user can complete each step.

**When to use**:
- Evaluate learnability (first-time use)
- New user onboarding flows
- Complex multi-step tasks

**Duration**: 1-2 hours per task

**Sample size**: 2-3 evaluators

**Process**:
1. Define user persona and goal
2. Define task steps
3. For each step, ask:
   - Will user know what to do?
   - Will user see how to do it?
   - Will user understand response?
   - Will user know they made progress?
4. Document issues and questions
5. Recommend improvements

**Best for**: Attitudinal (expert), qualitative, scripted use

**Deliverable**: Step-by-step issues, learnability problems, recommendations

**Pro tip**: Use realistic personas. Focus on new users, not experts.

---

### 8. A/B Testing

**What**: Quantitative comparison of two design variants to determine which performs better.

**When to use**:
- Optimize specific elements (buttons, copy, layout)
- Make data-driven design decisions
- Validate design changes

**Duration**: 1-4 weeks (depends on traffic)

**Sample size**: Calculate based on effect size (use A/B calculator)

**Process**:
1. Define hypothesis and success metric
2. Create variant A and variant B
3. Randomly assign users to variants
4. Collect data until statistical significance
5. Analyze results and implement winner

**Best for**: Behavioral, quantitative, natural use (live product)

**Deliverable**: Winning variant, conversion lift, statistical confidence

**Pro tip**: Test one change at a time. Ensure sufficient sample size before concluding. Watch for novelty effects.

## Measurement & Benchmarking Methods

### 9. Surveys

**What**: Structured questionnaires to collect attitudinal and behavioral data at scale.

**When to use**:
- Measure satisfaction, preferences, behaviors
- Quantify how widespread qualitative findings are
- Benchmark and track over time

**Duration**: 1-2 weeks (design, deploy, collect)

**Sample size**: 100+ for basic patterns, 400+ for segmentation

**Process**:
1. Define research questions
2. Design survey (mix of rating scales, multiple choice, open-ended)
3. Pilot test with 3-5 people (cognitive walkthrough + usability test)
4. Deploy to target population
5. Send reminders after 3-5 days
6. Analyze quantitative and qualitative data

**Best for**: Attitudinal, quantitative/qualitative, scripted use

**Deliverable**: Statistical findings, satisfaction scores, segmentation, quotes

**Pro tip**: Keep short (<10 min). Test thoroughly before deploying. Offer incentive if possible.

**Survey testing methods**:
1. **Cognitive walkthrough**: Test question clarity with users
2. **Mechanical test**: Check skip logic, branching
3. **Usability test**: Observe people taking survey

---

### 10. System Usability Scale (SUS)

**What**: 10-item standardized questionnaire measuring perceived usability.

**When to use**:
- Benchmark usability against industry norms
- Track usability over time
- Compare versions or competitors

**Duration**: 2 minutes per participant

**Sample size**: 100+ for reliable benchmark

**Process**:
1. Have users complete SUS after using product
2. Calculate SUS score (0-100 scale)
3. Interpret: 68 = average, >80 = excellent, <50 = poor
4. Compare to benchmarks or track over time

**Best for**: Attitudinal, quantitative, scripted use

**Deliverable**: SUS score, benchmark comparison

**Pro tip**: Use after usability testing or paired with qualitative research. SUS alone doesn't tell you *what* to fix.

---

### 11. Single Ease Question (SEQ)

**What**: Single question measuring task difficulty after each task.

**When to use**:
- Quick task-level usability metric
- Paired with usability testing
- Track task ease over time

**Duration**: 10 seconds per task

**Sample size**: 5+ per task

**Process**:
1. After each task in usability test, ask:
   "Overall, how difficult or easy was this task?" (1-7 scale)
2. Calculate average SEQ per task
3. Identify difficult tasks (SEQ < 4)

**Best for**: Attitudinal, quantitative, scripted use

**Deliverable**: Task difficulty ratings, comparison across tasks

**Pro tip**: Use immediately after each task. Combines well with task success rates.

---

### 12. Analytics Analysis

**What**: Analyze behavioral data from product usage (clicks, navigation, flows, time, etc.).

**When to use**:
- Understand actual behavior at scale
- Identify popular/unpopular features
- Find where users drop off
- Validate qualitative findings

**Duration**: 1-2 days (query, analyze, visualize)

**Sample size**: All users (entire dataset)

**Process**:
1. Define questions (what are you trying to learn?)
2. Query analytics (Google Analytics, Mixpanel, etc.)
3. Analyze patterns (flows, funnels, heatmaps)
4. Visualize findings
5. Combine with qualitative research (analytics shows *what*, research shows *why*)

**Best for**: Behavioral, quantitative, natural use

**Deliverable**: Usage patterns, drop-off points, feature popularity, trends

**Pro tip**: Analytics shows *what* but not *why*. Always pair with qualitative methods.

## Generative & Strategic Methods

### 13. Card Sorting

**What**: Participants organize topics into categories to understand mental models.

**When to use**:
- Design or evaluate information architecture (IA)
- Understand how users categorize information
- Name categories or navigation

**Duration**: 30-45 minutes per session

**Sample size**: 15-30 participants

**Types**:
- **Open card sort**: Participants create own categories
- **Closed card sort**: Participants use predefined categories

**Process**:
1. Create cards (topics/content items)
2. Ask participants to group cards
3. Ask participants to name groups (open sort)
4. Analyze patterns (similarity matrix, dendrograms)
5. Use findings to inform IA

**Best for**: Attitudinal, qualitative/quantitative, scripted use

**Deliverable**: Mental model map, IA recommendations, category names

**Pro tip**: Use online tools (Optimal Workshop, UserZoom) for larger samples. Analyze statistically for patterns.

---

### 14. Tree Testing

**What**: Evaluate findability by having users locate items in navigation structure (without visual design).

**When to use**:
- Validate information architecture
- Test navigation structure before design
- Compare IA alternatives

**Duration**: 15-20 minutes per session

**Sample size**: 50-100 participants (quantitative)

**Process**:
1. Define tasks (find specific items)
2. Present text-only navigation hierarchy
3. Ask users to find items
4. Track: success, path, time
5. Identify problem areas (low success rates, wrong paths)

**Best for**: Behavioral, quantitative, scripted use

**Deliverable**: Findability metrics, navigation issues, IA recommendations

**Pro tip**: Test before visual design. Compare alternative IAs. Use online tools (Optimal Workshop).

---

### 15. Focus Groups

**What**: Moderated group discussion (6-8 people) about attitudes and experiences.

**When to use**:
- Generate ideas and reactions
- Understand group dynamics and social context
- Explore attitudes and motivations

**Duration**: 1.5-2 hours per group

**Sample size**: 3-5 groups of 6-8 people each

**Process**:
1. Recruit homogeneous groups (similar backgrounds)
2. Create discussion guide
3. Moderate discussion (encourage all to participate)
4. Observe group dynamics
5. Analyze themes and attitudes

**Best for**: Attitudinal, qualitative, scripted use

**Deliverable**: Themes, attitudes, group perspectives, reactions

**Pro tip**: Don't use for individual tasks or behaviors - use for attitudes. Watch for groupthink. Skilled moderation essential.

---

### 16. Participatory Design / Co-Design

**What**: Collaborate with users to design solutions together.

**When to use**:
- Design for complex or specialized domains
- Ensure user needs are central
- Generate ideas with users

**Duration**: 2-4 hours per workshop

**Sample size**: 4-8 users per session, multiple sessions

**Process**:
1. Define design challenge
2. Recruit representative users
3. Facilitate workshop (sketching, brainstorming, prototyping)
4. Users create design concepts
5. Synthesize and refine concepts

**Best for**: Attitudinal/behavioral, qualitative, scripted use

**Deliverable**: Design concepts, user insights, requirements

**Pro tip**: Provide materials (paper, markers, sticky notes). Facilitate, don't dictate. Users are experts in their needs.

## Advanced & Specialized Methods

### 17. Eye Tracking

**What**: Track where users look on interface using specialized hardware.

**When to use**:
- Understand visual attention
- Evaluate visual hierarchy
- Optimize layouts and content placement

**Duration**: 30-60 minutes per session (plus calibration)

**Sample size**: 10-15 users

**Process**:
1. Calibrate eye tracker for participant
2. Present interface or design
3. Give tasks or free exploration
4. Analyze gaze patterns, heatmaps, fixations
5. Identify attention problems

**Best for**: Behavioral, quantitative/qualitative, scripted use

**Deliverable**: Heatmaps, attention patterns, design recommendations

**Pro tip**: Expensive and time-intensive. Use when visual attention is critical (e.g., ads, complex dashboards).

---

### 18. First Click Testing

**What**: Measure where users first click to complete a task.

**When to use**:
- Evaluate navigation clarity
- Test whether users can find starting point
- Quick validation of designs

**Duration**: 5-10 minutes per session

**Sample size**: 30-50 participants

**Process**:
1. Present static design or wireframe
2. Give task (e.g., "Where would you click to...")
3. Capture first click location
4. Analyze: Is first click correct? How many different clicks?
5. Identify confusing areas

**Best for**: Behavioral, quantitative, scripted use

**Deliverable**: First click success rates, confusion points

**Pro tip**: First click predicts task success. If first click is wrong, users rarely recover.

---

### 19. Five-Second Test

**What**: Show design for 5 seconds, then ask what users remember.

**When to use**:
- Test first impressions
- Evaluate visual hierarchy and messaging
- Validate landing pages or key screens

**Duration**: 5 seconds exposure + 2 minutes questions

**Sample size**: 20-30 participants

**Process**:
1. Show design for 5 seconds
2. Hide design
3. Ask: What do you remember? What was it for? What could you do?
4. Analyze recall patterns
5. Identify clarity issues

**Best for**: Attitudinal, qualitative/quantitative, scripted use

**Deliverable**: Recall metrics, clarity issues, messaging effectiveness

**Pro tip**: Tests first impressions only. Use for initial clarity, not detailed usability.

---

### 20. Remote Unmoderated Testing

**What**: Users complete tasks on their own, remotely, while software records session.

**When to use**:
- Quick feedback at scale
- Access geographically distributed users
- Lower cost than moderated testing

**Duration**: 15-20 minutes per participant (self-guided)

**Sample size**: 10-30 participants

**Process**:
1. Define tasks and questions
2. Set up in remote testing platform (UserTesting, Maze, etc.)
3. Recruit participants through platform
4. Participants complete tasks asynchronously
5. Review recordings and analyze

**Best for**: Behavioral, qualitative/quantitative, scripted use

**Deliverable**: Task videos, completion metrics, user feedback

**Pro tip**: Can't probe or clarify. Tasks must be very clear. Good for broad feedback, not deep investigation.

## Method Selection Decision Matrix

### By Research Phase

| Phase | Methods |
|-------|---------|
| **Discover** | User Interviews, Contextual Inquiry, Diary Studies, Field Studies |
| **Explore** | Card Sorting, Focus Groups, Participatory Design |
| **Evaluate** | Usability Testing, Heuristic Evaluation, Cognitive Walkthrough, A/B Testing |
| **Measure** | Surveys, SUS, SEQ, Analytics, Eye Tracking |
| **Validate** | Tree Testing, First Click, Five-Second Test, Remote Testing |

---

### By Timeline

| Timeline | Methods |
|----------|---------|
| **1-2 days** | Heuristic Evaluation, Analytics Analysis, Expert Review |
| **3-7 days** | Usability Testing (5 users), First Click, Five-Second Test |
| **1-2 weeks** | User Interviews, Surveys (quick), Remote Testing, Card Sorting |
| **2-4 weeks** | Contextual Inquiry, Surveys (comprehensive), Tree Testing, Focus Groups |
| **1+ months** | Diary Studies, Field Studies, Longitudinal A/B Tests |

---

### By Sample Size

| Sample Size | Methods |
|-------------|---------|
| **< 10** | User Interviews, Contextual Inquiry, Usability Testing, Expert Evaluation |
| **10-30** | Card Sorting, Eye Tracking, First Click, Five-Second Test |
| **30-100** | Tree Testing, Remote Testing |
| **100+** | Surveys, SUS, A/B Testing, Analytics |

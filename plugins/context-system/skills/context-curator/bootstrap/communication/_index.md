## When to Use This Domain

**Always load**: Communication preferences are foundational and apply to every interaction with Claude.

**Update when**:
- You notice patterns in corrections
- Your preferences change over time
- You discover new preferences
- Feedback style isn't working
- Communication feels misaligned

**What to document**:
- **Response style**: Review-first vs execute-first
- **Explanation depth**: Minimal, moderate, detailed, adaptive
- **Tone & voice**: Professional, conversational, direct
- **Emoji usage**: Never, minimal, moderate, frequent
- **Feedback style**: Validation vs critique focused
- **Format preferences**: Code formatting, documentation style

## Getting Started

**First time**: Open [[communication/style-preferences]] and fill in your preferences using the template provided.

**Examples to help**:
- Review-first: "Propose changes before executing"
- Execute-first: "Make changes directly, show results"
- Detailed explanations: "Include reasoning and trade-offs"
- Critique-focused: "Point out problems, not just validation"

## When to Use This Domain

**Load when**:
- Planning multi-step tasks (3+ steps)
- Breaking down complex problems
- Creating execution roadmaps
- Organizing project workflows
- Validating plan quality
- Managing task dependencies

**Key practices**:
- Phased planning (decompose → validate → execute)
- Hierarchical decomposition (goals → milestones → tasks → actions)
- Quality validation (completeness, feasibility, efficiency)
- Evidence-based planning approaches
